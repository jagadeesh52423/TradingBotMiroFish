ROOT CAUSE ANALYSIS
───────────────────

Why 1.71M alerts were generated:
  1. No setup-level deduplication
     → Each orderflow tick re-evaluates alerts
     → Identical (symbol, direction, entry) generates 1,000+ alerts
  
  2. No cooldown mechanism
     → Same setup can alert multiple times
     → No prevention of repeat signals
  
  3. Low confidence threshold (65%)
     → Many weak signals included
     → 65-70% confidence has high false positive rate
  
  4. Alert engine design flaw
     → Alerts per tick instead of per setup
     → No aggregation or consolidation logic

═══════════════════════════════════════════════════════════════════════════════

SOLUTION APPLIED
────────────────

✓ Setup-level grouping (120s window)
  → Combines identical (symbol, direction, entry) into one alert
  → Takes best (highest confidence) representative

✓ 10-minute cooldown
  → Prevents same setup from alerting multiple times
  → Requires 600+ seconds before re-alerting

✓ >= 70% confidence threshold
  → Eliminates low-signal noise (65-70% range)
  → Keeps only moderate-to-high confidence signals

✓ Regime + displacement + reason code validation
  → Ensures only valid market structure signals
  → Confirms sweep_detected or absorption present
  → Verifies meaningful price displacement
